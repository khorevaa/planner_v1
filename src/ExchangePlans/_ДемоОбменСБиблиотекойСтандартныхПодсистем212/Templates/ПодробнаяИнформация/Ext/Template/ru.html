Синхронизация данных с Демо: 1С:Библиотека стандартных подсистем 8.2, версия 2.1.2 предназначена для демонстрации обратной совместимости правил обмена между конфигурациями старшей и младшей версий. Синхронизация данных является демонстрационной и не предназначена для работы с реальными прикладными данными. Выполняется синхронизация следующих данных:

  • Справочник "Демо: Номенклатура"
  • Справочник "Организации" - выполняется синхронизация контактной информации. При отправке данных в версию 2.1.2 выполняется конвертация значения реквизита ЗначенияПолей в предыдущий формат (см. функцию ПредыдущийФорматКонтактнойИнформацииXML модуля УправлениеКонтактнойИнформацией). При обратном приеме данных из версии 2.1.2 конвертация не выполняется, поскольку данные корректно загружаются и интерпретируются в предыдущем формате.
  • Справочник "Страны мира" - элементы справочника синхронизируются по ссылке с дополнительным сопоставлением по коду и наименованию. Поскольку пользователи могут заполнять справочник произвольными странами (например, таможенными союзами для учета стран происхождения товара), которых нет в классификаторе стран мира (ОКСМ), то целевой объект сначала ищется по ссылке (если он был введен в одной из программ и был ранее синхронизирован), а затем, в случае неудачи, по составному ключу "Код" + "Наименование" (если выполняется синхронизация стран, введенных вручную или загруженных из классификатора независимо в обоих программах).
  • Документ "Демо: Заказ покупателя" - без потерь данных конвертируются реквизиты, которые различаются в текущей и предыдущей версиях конфигурации. Реквизит СтатусЗаказа синхронизируется с устаревшим реквизитом ЗаказЗакрыт (БСП 2.1.2 и ранее). Кроме того, при получении данных из версии 2.1.2, табличная часть КонтактнаяИнформация не очищается (так как в предыдущей версии этой табличной части не было), а корректируется в соответствии с новыми данными полученной табличной части ПартнерыИКонтактныеЛица. Также корректируется и табличная часть ПартнерыИКонтактныеЛица - автоматические вычисляется реквизит ИдентификаторСтрокиТабличнойЧасти, которого не было в предыдущей версии.

При настройке можно также указать организации, по которым будут синхронизироваться данные. Также при настройке можно задать дату, начиная с которой будут синхронизироваться заказы покупателей.


Для демонстрации синхронизации данных с версией 2.1.2 в обе стороны необходимо развернуть вторую информационную базу версии 2.1.2, взяв ее из дистрибутива - ExtFiles\Обмены данными\Библиотека стандартных подсистем, версия 2.1.2\1Cv8.dt
