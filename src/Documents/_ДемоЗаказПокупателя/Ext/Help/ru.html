Документ предназначен для примера использования подсистем Бизнес-процессы и задачи, Даты запрета изменения, Веб-сервисы работы с файлами, Взаимодействия, Запрет редактирования реквизитов объектов, Префиксация объектов, Присоединенные файлы, Работа с файлами, Структура подчиненности, Управление доступом.

В подсистеме Бизнес-процессы и задачи на основании документа можно ввести бизнес-процесс Задание, он также может выступать предметом задач.

В подсистеме Даты запрета изменения документ показывает пример запрета изменения документа по разделу дат запрета Демо: Торговля с объектом Демо: Группа доступа партнеров, который получается, как реквизит Демо: Группа доступа покупателей объекта Демо: Партнер, заданного в реквизите Партнер документа Демо: Заказ покупателя.

В подсистеме Веб-сервисы работы с файлами документ показывает пример объекта, при котором хранятся вложения в другой информационной базе.

В подсистеме Взаимодействия документ выступает предметом взаимодействий.

В подсистеме Запрет редактирования реквизитов объектов документ показывает пример того, как реквизиты Организация, Партнер и табличная часть СчетаНаОплату могут быть заблокированы для изменения.

В подсистеме Префиксация объектов к коду элементов справочника добавляется префикс информационной базы.

В подсистеме Присоединенные файлы документ используется в качестве примера объекта-владельца файлов вложений.

В подсистеме Работа с файлами документ показывает пример объекта-владельца файлов вложений.

В подсистеме Структура подчиненности документ показывает пример иерархической взаимосвязи документов (реквизит Счет табличной части СчетаНаОплату ссылается на документ Демо: Счет на оплату покупателю).

В подсистеме Управление доступом документ показывает пример реализации ограничения в ролях, содержащего два вида доступа Демо: Организации (для реквизита Организация) и Демо: Группы партнеров (для реквизита Партнер). Кроме этого, документ записывает набор значений доступа, который используется бизнес-процессом Демо: Задание с ролевой адресацией.
