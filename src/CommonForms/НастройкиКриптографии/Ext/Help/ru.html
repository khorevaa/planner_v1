Настройка параметров для использования средств криптографии: электронной подписи (ЭП) и шифрования.

Открывается по команде Настройки криптографии раздела Администрирование - Настройки работы с файлами.

Для работы с криптографией вам потребуется установить стороннюю программу - криптопровайдер.

Для того чтобы использовать ЭП, необходимо включить соответствующий флажок в разделе Администрирование - Настройки работы с файлами. После этого становится доступной команда Настройки криптографии.

Электронная подпись - реквизит любого электронного документа, с помощью которого можно удостовериться в неизменности электронного документа с момента его подписи. Для подписи необходимо иметь сертификат ключа ЭП. Сертификат ключа ЭП можно получить с помощью криптографических сервисов (провайдеров) или операционной системы.

До начала использования ЭП следует определить, какие ЭП будут использоваться в программе: юридически значимые или нет.

Для юридически значимых ЭП необходимо приобрести специальную программу - криптопровайдер, поддерживающий ГОСТ Р 34.10-2001, например, Крипто Про, и установить его на компьютеры с программой 1С:Предприятие. Кроме того, потребуется заключить договор с удостоверяющим центром - для получения сертификатов.

Если юридически значимая ЭП не нужна (например, для использования ЭП только внутри предприятия), то можно использовать любой криптопровайдер, например, Microsoft Enhanced Cryptographic Provider v1.0. Указанный криптопровайдер входит в состав ОС Windows и его не нужно приобретать отдельно. Сертификаты можно получить с помощью домена контроллера Windows Server 2003 или 2008 (с помощью утилиты certmgr.msc, выполнив команду All tasks - Request new certificate). Более подробно описание утилиты см. справку к ОС Windows, а также сайт фирмы Microsoft http://technet.microsoft.com/en-us/library/cc754122%28WS.10%29.aspx.

В случае использования распределенной информационной базы эти настройки могут быть изменены только в главном узле.

Настройки криптографии

  • Провайдер ЭП - строка, идентифицирующая провайдера ЭП, например: "Crypto-Pro GOST R 34.10-2001 Cryptographic Service Provider" - идентификатор криптографического провайдера от компании Крипто Про.
    • Ниже перечислены некоторые наиболее распространенные криптопровайдеры (название, имя и тип криптопровайдера для указания в диалоге настроек криптографии):
      • КриптоПро CSP:   "Crypto-Pro GOST R 34.10-2001 Cryptographic Service Provider"  (тип 75);
      • Signal-COM CSP:  "Signal-COM CPGOST Cryptographic Provider" (тип 75);
      • ViPNet CSP: "Infotecs Cryptographic Service Provider" (тип 2);
      • Microsoft CSP: "Microsoft Enhanced Cryptographic Provider v1.0" (тип 1).
    • Если вашего криптопровайдера нет в этом списке, вам надо связаться с фирмой-производителем криптопровайдера и получить необходимую информацию.
  • Тип провайдера ЭП - число, указывающее тип провайдера ЭП. Например, для криптопровайдеров, поддерживающих ГОСТ Р 34.10-2001, выберите тип 75.
  • Алгоритм подписи - выберите значение из списка, предоставляемого провайдером ЭП.
  • Алгоритм хеширования - выберите значение из списка, предоставляемого провайдером ЭП.
  • Алгоритм шифрования - выберите значение из списка, предоставляемого провайдером ЭП.
  • Выполнять проверку ЭП на сервере - если флажок установлен, то проверка ЭП выполняется на сервере, иначе - на клиенте.
  • Путь модуля криптографии (для сервера 1С:Предприятия под управлением Linux) - путь к модулю криптографии для операций с файлами (расшифровка, подписание, проверка подписи, просмотр сертификата). Данная настройка становится доступной, если установлен флажок Выполнять проверку ЭП на сервере. Необходимо заполнить параметры:
    • Имя компьютера - напишите сетевое имя компьютера;
    • Путь модуля криптографии - напишите путь к модулю криптографии на этом компьютере;
    • Комментарий - напишите дополнительные сведения.
